Q1: What is the difference between global and local variables in Python?

Answer:

  • â€¢ Global variables: Defined outside functions, accessible throughout the program
  • â€¢ Local variables: Defined inside functions, only accessible within that function
  • â€¢ Use global keyword to modify global variables inside functions
  • â€¢ Local variables shadow global variables with the same name

def function():
    global x
    x = 20  # Modifies global x
    y = 30  # Local variable

Q3: What happens when you assign one variable to another in Python?

Answer:

It depends on whether the object is mutable or immutable:

Immutable Objects (int, str, tuple):

Creates a reference to the same object initially, but any modification creates a new object

Mutable Objects (list, dict, set):

Creates a reference to the same object; modifications affect both variables

Coding Q1: Variable Swapping Challenge

Problem: Given two variables a=10 and b=20, swap their values using at least 3 different methods. Explain the pros and cons of each method.

Solution:

# Method 1: Using temporary variable (Universal)
a, b = 10, 20
print(f"Before: a={a}, b={b}")
temp = a
a = b
b = temp
print(f"Method 1: a={a}, b={b}")

# Method 2: Tuple unpacking (Pythonic)
a, b = 10, 20
a, b = b, a
print(f"Method 2: a={a}, b={b}")

# Method 3: Arithmetic operations (Numbers only)
a, b = 10, 20
a = a + b  # a = 30
b = a - b  # b = 10
a = a - b  # a = 20
print(f"Method 3: a={a}, b={b}")

# Method 4: XOR operations (Integers only)
a, b = 10, 20
a = a ^ b
b = a ^ b
a = a ^ b
print(f"Method 4: a={a}, b={b}")

# Method 5: Using list (Creative but inefficient)
a, b = 10, 20
swap_list = [a, b]
a, b = swap_list[1], swap_list[0]
print(f"Method 5: a={a}, b={b}")
Comparison:

Best: Tuple unpacking - clean, readable, Pythonic

Universal: Temporary variable - works with any data type

Risky: Arithmetic - can cause overflow

Limited: XOR - integers only

Coding Q2: Reference vs Copy Problem

Problem: Predict the output of this code and explain why: 

list1 = [1, 2, [3, 4]]
list2 = list1
list3 = list1.copy()
list4 = list1[:]
import copy
list5 = copy.deepcopy(list1)

list1[0] = 'X'
list1[2].append(5)

print("list1:", list1)
print("list2:", list2)
print("list3:", list3)
print("list4:", list4)
print("list5:", list5)
Answer & Explanation:

list1: ['X', 2, [3, 4, 5]] - Original, modified

list2: ['X', 2, [3, 4, 5]] - Reference to list1, same object

list3: [1, 2, [3, 4, 5]] - Shallow copy, nested list is shared

list4: [1, 2, [3, 4, 5]] - Slice copy, same as shallow copy

list5: [1, 2, [3, 4]] - Deep copy, completely independent

Coding Q3: Variable Scope Tricky Question

Problem: What will this code print and why? 

x = 10

def func1():
    print(f"func1: {x}")

def func2():
    x = 20
    print(f"func2: {x}")

def func3():
    print(f"func3 before: {x}")
    x = 30
    print(f"func3 after: {x}")

func1()
func2()
try:
    func3()
except Exception as e:
    print(f"Error: {e}")
Answer:

func1: 10 - Accesses global x

func2: 20 - Creates local x, shadows global

func3: UnboundLocalError - Python sees assignment to x, treats it as local, but tries to print before assignment

Fix: Add global x at the beginning of func3

Q: How does Python's garbage collection affect variable lifecycle?

Answer:

  • â€¢ Python uses reference counting + cyclic garbage collection
  • â€¢ Variables are cleaned when reference count reaches zero
  • â€¢ Circular references need cyclic GC to be collected
  • â€¢ weakref module provides weak references
  • â€¢ gc module allows manual control

Q: Implement a variable that automatically logs all access and modifications.

class LoggedVariable:
    def __init__(self, value, name="variable"):
        self._value = value
        self._name = name
        self._access_log = []
    
    def __get__(self, instance, owner):
        self._access_log.append(f"READ: {self._value}")
        return self._value
    
    def __set__(self, instance, value):
        old_value = self._value
        self._value = value
        self._access_log.append(f"WRITE: {old_value} -> {value}")
    
    def get_log(self):
        return self._access_log.copy()
